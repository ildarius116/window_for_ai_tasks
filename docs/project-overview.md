# MWS GPT Platform - Project Overview

## High-Level Architecture
The MWS GPT Platform is a unified AI workspace combining text, voice, image, file processing, and long-term memory capabilities using a microservices Docker architecture.

## Key Components

### 1. Backend (FastAPI)
**Location:** `mws-gpt/backend/`
- **Structure:**
  - `app/main.py` - Application factory
  - `app/api/v1/` - REST routers (auth, chat, memory, files, voice, models, admin)
  - `app/api/websockets/` - Real-time chat streaming
  - `app/core/` - Business logic (AI router, auth, file processor, memory service, RAG, voice)
  - `app/models/` - SQLAlchemy ORM models
  - `app/repositories/` - Data access layer
  - `app/schemas/` - Pydantic validation schemas
  - `app/tasks/` - Celery background tasks
- **Key Technologies:** Python 3.14+, FastAPI, SQLAlchemy, Pydantic, Celery, Redis, pgvector

### 2. Frontend (React + OpenWebUI Fork)
**Location:** `mws-gpt/frontend/src/`
- Minimal template structure with components/chat, fileupload, modelselector, voice

### 3. LiteLLM AI Gateway
**Location:** `mws-gpt/litellm/`
- Unified proxy for OpenAI, Anthropic, Ollama providers
- Routes requests, handles auth, rate limiting, cost tracking
- Listens on port 4000

### 4. Database Layer
- **PostgreSQL 16 + pgvector** (Port 5432) - Primary data store with vector extensions
- **Alembic** - Database migrations

### 5. Caching & Message Broker
- **Redis 7** (Port 6379) - Session storage, Celery broker

### 6. Background Processing
- **Celery Workers** - Async file embedding, memory summarization
- **Flower** (Port 5555) - Celery monitoring UI

### 7. Monitoring & Observability
- **Prometheus** (Port 9090) - Metrics collection
- **Grafana** (Port 3002) - Visualization dashboards
- **Langfuse** (Port 3001) - LLM usage tracing
- **Nginx** - Reverse proxy (Ports 80/443)

### 8. Local AI Support (Optional)
- **Ollama** (Port 11434) - Local LLM hosting
