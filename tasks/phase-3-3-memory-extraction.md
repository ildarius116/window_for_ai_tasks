# Phase 3.3 — Memory Extraction: извлечение фактов из разговора

**Агент:** MemoryAgent
**Зависимости:** 3.2
**Статус:** TODO

## Задача

LLM-powered извлечение важных фактов из завершённого разговора и сохранение в Memory Service.

## Логика

После каждого разговора (хук или scheduled job):
1. Взять последние N сообщений разговора
2. Отправить в LiteLLM Sonnet с промптом-экстрактором
3. Получить список фактов (JSON)
4. Дедуплицировать через семантический поиск (если факт уже есть — не добавлять)
5. Сохранить новые факты

## Промпт-экстрактор

```
Из следующего диалога извлеки важные факты о пользователе:
- Предпочтения (языки программирования, фреймворки, стиль)
- Контекст работы (проект, роль, технологии)
- Явные просьбы ("всегда делай X", "я предпочитаю Y")
- Ключевые договорённости

Верни JSON: {"memories": ["факт 1", "факт 2"]}
Только реально важные факты. Максимум 5 за разговор.
```

## Где вызывать

Вариант A: OpenWebUI pipeline (после каждого ответа)
Вариант B: Cron job каждые 30 мин — обрабатывать незаписанные разговоры

Начать с Вариантом A.

## Критерии готовности

- [ ] Extraction pipeline вызывается после разговора
- [ ] Факты реально извлекаются (не заглушка)
- [ ] Дедупликация работает (нет дублей при повторном запуске)
- [ ] Факты сохраняются в pgvector
